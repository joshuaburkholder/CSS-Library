Styling Methods

Inline Styling

It is possible to style each HTML element with inline styling. For example,

<[element name] STYLE="[CSS property]:[property value]">

However, this is not always the best approach. In order to make changes, one would have to go through the entire HTML markup and change the styling for each individual element.

This approach also does not give allow you to use all the wonders of CSS. A better approach would be to only use inline styling for overriding styling rules on a case-by-case basis. (Or just don't do it. Seriously.)

Enter CSS!

CSS (Cascading Style Sheets) allow web developers to separate the design aspects of a web site from the HTML markup.

This is considered a best practice in development, and there are a couple of different approaches.

Embedded Styling

Embedded styling allows the presentation rules to be embedded into the HTML <head>.

Notice the "presentation rule" located in the <head> tag, just under the <title>. You'll see the following code, which gives us this great grey background:

<style> body{background: #d3d3d3}</style>

This method allows you to access to all the wonders of CSS, but it's application is limited to a single HTML page.

External Style Sheets

Extneral style sheets create a cleaner way to approach designing a webpage and also allow you to create consistency in design across multple pages.

In the <head> tag, following the &/style> tag, you'll see the following code:

<link rel="stylesheet" type="text/css" href="intro.css">

This links the HTML file to an external style sheet, intro.css. You'll notice that is where we are defining properties for div, h2, etc. This method gives us access to the full-range of CSS capabilities, as well as creates a clean way for us to apply styling rules across multiple pages.

Higher Precedence

One important thing to consider is the precedence given to these methods. Because specificity matters in how styling is rendered (more on that later), embedded styles have a higher precedence than external styles. So, if there are equivalent rules given, the embedded style will take precedence over the external. Likewise, inline styling will have a higher precedence over embedded styling.

Order of precedence, from highest to lowest

Inline Styling > Embedded Styling > External Style Sheet
