Why CSS?

Inline Styling

It is possible to style each HTML element with inline styling. For example,

<[element name] STYLE="[CSS property]:[property value]">

However, this is not always the best approach. In order to make changes, one would have to go through the entire HTML markup and change the styling for each individual element.

A better approach would be to only use inline styling for overriding styling rules on a case-by-case basis. (Or just don't do it. Seriously.)

Enter CSS!

CSS (Cascading Style Sheets) allow web developers to separate the design aspects of a web site from the HTML markup.

This is considered a best practice in development, and there are a couple of different approaches.

Embedded Styling

Embedded styling allows the presentation rules to be embedded into the HTML <head>.

Notice the "presentation rule" located in the <head> tag, just under the <title>. You'll see the following code, which gives us this great grey background:

<style> body{background: #d3d3d3}</style>
